5. AMBIENTE

Ambiente Ideal

O ambiente ideal para rodar o sistema seria algo mais robusto, como um servidor dedicado, banco de dados com backups automáticos e uma rede estável.


Funcionalidades Pendentes

Seria ideal que o sistema oferecesse as seguintes funções:

  • Possibilidade do cliente acompanhar o status de conserto do aparelho através de mensagens automáticas na forma de contato definida por ele, e-mail ou Whatsapp.
  • Integração com formas de pagamento, permitindo o recebimento direto de pagamentos diretamente pelo sistema.
  • Cadastro de aparelho com foto, registrando o estado do aparelho no momento da entrada (tela quebrada, riscos, etc.), evitando conflitos na retirada.
  • Gerenciamento de saídas financeiras para registrar e acompanhar as despesas com compras de produtos ou outros materiais.
  • Cadastro de fornecedores com dados de contato, histórico de compras e controle de prazos de entrega, facilitando a reposição de produtos.
  • Módulo de Fidelidade que contabiliza pontos que serão trocados por descontos para clientes recorrentes.
  • Dashboard para exibição visual e resumida com os principais indicadores do sistema.
  • Relatórios mais avançados, com gráficos e painéis interativos mostrando faturamento por período, produtos mais usados, serviços mais realizados e desempenho da equipe.
  • Aplicativo Mobile nativo para técnicos e clientes.
  • Integração com Redes Sociais com compartilhamento de status e promoções.
  • Sistema de Agendamento com marcação de horários para atendimento.

Os elementos acima serão avaliados para integração nas próximas versões do sistema.


Além disso, a hospedagem ideal envolveria servidores dedicados, banco de dados com backup automático e suporte técnico ativo, garantindo maior desempenho, segurança e disponibilidade.

Ambiente Proposto

Considerando as limitações de recursos e a necessidade de uma solução acessível para pequenas lojas de conserto, o sistema proposto foca nas funcionalidades essenciais para o funcionamento básico do negócio.


Infraestrutura Simplificada

O ambiente proposto será implementado com:

  • Hospedagem gratuita através do Azure for Students
  • Banco de dados SQL Server básico
  • Interface web responsiva acessível via navegador
Funcionalidades Principais
  • Cadastro de clientes com duas opções de tipos de cadastro (simples ou completo).
  • Criação e acompanhamento de ordens de serviços registrando os serviços solicitados, status, valores, formas de pagamento, prazos, atendentes e técnicos responsáveis.
  • Cadastro e controle de produtos com baixa automática no estoque.
  • Gestão financeira com o controle de entradas de pagamentos.
  • Histórico de atendimento de serviços realizados para cada cliente.
  • Geração de relatórios personalizáveis em PDF com base em filtros diversos.
  • Mecanismos de busca e filtros para localização rápida de informações.
  • Logs de Auditoria com histórico de alterações com identificação de usuário e timestamp.
  • Cadastro de funcionários e usuários com níveis diferenciados de autoridade e acesso.
